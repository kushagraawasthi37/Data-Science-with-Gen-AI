====================================================
1. WHAT IS A WEB FRAMEWORK?
====================================================

Web framework ek ready-made toolkit hota hai jo web applications banana easy bana deta hai.

Without framework:
- HTTP requests manually handle karni padti
- Routing khud likhni padti
- Security, sessions, cookies sab khud
- Database integration complex hota

With framework:
- Routing already available
- Template engine milta hai
- Security features built-in
- Database ke tools milte hain

Interview line:
A web framework abstracts low-level web complexities and allows developers to focus on business logic.

====================================================
2. TYPES OF WEB FRAMEWORKS
====================================================

1. Full / Macro Frameworks
Examples:
- Django
- Ruby on Rails

Features:
- ORM built-in
- Authentication
- Admin panel
- Heavy but powerful

2. Micro Frameworks
Examples:
- Flask
- Bottle
- CherryPy

Features:
- Lightweight
- Sirf basic features
- Developer decide karta hai kya add karna hai

Interview tip:
Flask microframework hai kyunki yeh sirf essential tools provide karta hai.

====================================================
3. WHAT IS FLASK?
====================================================

Flask ek lightweight Python web framework hai.

Creator:
Armin Ronacher
Year: 2010

Flask internally use karta hai:
- WSGI (Web Server Gateway Interface)
- Werkzeug (request/response handling)
- Jinja2 (template engine)

Interview line:
Flask is built on Werkzeug and Jinja2.

====================================================
4. FLASK VS DJANGO
====================================================

Flask:
- Micro framework
- ORM nahi hota
- Zyada flexible
- APIs & microservices ke liye best

Django:
- Full framework
- ORM built-in
- Admin panel
- Large applications ke liye best

Interview conclusion:
Full features chahiye -> Django
Lightweight & control chahiye -> Flask

====================================================
5. PROS AND CONS OF FLASK
====================================================

Advantages:
1. Easy to learn
2. Highly flexible
3. Built-in debugger
4. Simple routing & DB integration

Disadvantages:
1. Default async support nahi
2. Large applications ke liye limited

Interview analogy:
Flask ek single cashier jaisa hai, zyada traffic aaya to queue ban jaati hai.

====================================================
8. BASIC FLASK APPLICATION
====================================================

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello Flask"

if __name__ == "__main__":
    app.run(debug=True)

Explanation:
- Flask(__name__) creates app instance
- @app.route('/') maps URL
- debug=True enables auto reload & error page

====================================================
9. ROUTING IN FLASK
====================================================

Routing = URL ko Python function se map karna

Example:

@app.route('/courses')
def courses():
    return "Data Science Courses"

Multiple routes:

@app.route('/')
@app.route('/home')
def home():
    return "Home Page"

Interview tip:
Flask routing decorators ka use karta hai.

====================================================
12. REQUEST OBJECT
====================================================

Request object client se aayi request ka data rakhta hai:
- query params
- form data
- headers
- cookies

Example:

from flask import request

request.args.get('name')   (GET)
request.form.get('name')   (POST)

13. RESPONSE OBJECT
====================================================
